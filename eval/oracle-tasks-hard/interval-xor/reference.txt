Reasoning: For integer intervals, `[s, e]` covers `e - s + 1` points. The number of points covered by exactly one list equals `|A| + |B| - 2*|A∩B|`, where `|A|` and `|B|` are total covered points and `|A∩B|` is the count of integer points covered by both. Because each list is disjoint and sorted, I compute the intersection with a two-pointer merge: for the current pair, the shared integer points are `[max(sA,sB), min(eA,eB)]`, contributing `e - s + 1` points when `s <= e` (this correctly treats adjacent-but-not-touching intervals like `[1,3]` and `[4,6]` as zero overlap, and shared-endpoint intervals like `[1,3]` and `[3,6]` as 1). Advance the pointer whose interval ends first.

Checks by hand: identical lists → totals cancel → 0. `[[7,7]]` vs `[[9,9]]` → 1+1-0 = 2. `[[1,3]]` vs `[[3,6]]` → 3+4-2 = 5. nested `[[1,10]]` vs `[[3,5]]` → 10+3-6 = 7. empty side → 0+2-0 = 2.

```python
def symmetric_difference_length(A, B):
    def total(iv):
        return sum(e - s + 1 for s, e in iv)
    i = j = 0
    both = 0
    while i < len(A) and j < len(B):
        s = max(A[i][0], B[j][0]); e = min(A[i][1], B[j][1])
        if s <= e:
            both += e - s + 1
        if A[i][1] < B[j][1]:
            i += 1
        else:
            j += 1
    return total(A) + total(B) - 2 * both
```
