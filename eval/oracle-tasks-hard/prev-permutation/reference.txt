This is `prev_permutation`, the mirror of the standard `next_permutation`: every comparison is flipped, and the "already at the extreme" case wraps to the sorted-descending sequence.

Algorithm:
1. Scan from the right for the largest index `i` with `a[i] > a[i+1]` (i.e. skip the trailing ascending run). Use `a[i] <= a[i+1]` as the "keep scanning" test so duplicates are treated correctly.
2. If no such `i` exists, `a` is fully ascending (the first permutation) — reverse it to get the descending last permutation and return.
3. Otherwise find the largest `j > i` with `a[j] < a[i]`. Because the suffix after `i` is ascending, use `a[j] >= a[i]` as the "keep scanning" test (the `>=`, not `>`, is what makes repeated values correct).
4. Swap `a[i]` and `a[j]`, then reverse the suffix `a[i+1:]`.

Hand checks: `[2,1,3]` -> i=0, j=1, swap -> `[1,2,3]`, reverse suffix -> `[1,3,2]`. `[1,5,1]` -> i=1, j=2 -> `[1,1,5]`. `[3,1,3]` -> i=0, j=1 (a[1]=1<3), swap -> `[1,3,3]`, reverse `[3,3]` -> `[1,3,3]`. `[1,2,3,4]` -> no descent, reverse -> `[4,3,2,1]`.

```python
def prev_permutation(a):
    a = list(a)
    n = len(a)
    i = n - 2
    while i >= 0 and a[i] <= a[i + 1]:
        i -= 1
    if i < 0:
        a.reverse()
        return a
    j = n - 1
    while a[j] >= a[i]:
        j -= 1
    a[i], a[j] = a[j], a[i]
    a[i + 1:] = reversed(a[i + 1:])
    return a
```
