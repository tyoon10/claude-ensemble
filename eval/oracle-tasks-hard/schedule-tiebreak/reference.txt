Reasoning: This is interval scheduling for maximum COUNT, but conflicts include touching (shared endpoints), so compatibility requires a STRICT gap: an interval may follow a chosen one only if its start is strictly greater than the last chosen end. The canonical optimum sorts by end ascending, breaking ties by later start (this never reduces the count and pins down the unique answer), then by input index, and greedily selects while scanning. Finally we return the chosen intervals in chronological (start-ascending) order.

Walking the required examples: [[0,1],[1,2],[2,3],[3,4]] -> pick [0,1] (end 1), skip [1,2] (1 not > 1), pick [2,3], skip [3,4] (3 not > 3) -> [[0,1],[2,3]]. [[0,1],[3,7],[4,7]] -> pick [0,1], among the end-7 pair the later start 4 is tried first and 4 > 1 so pick [4,7] -> [[0,1],[4,7]]. [[1,4],[2,4],[3,4],[0,5]] -> the three end-4 intervals are ordered by later start first, so [3,4] is picked and everything else conflicts -> [[3,4]].

```python
def schedule(intervals):
    order = sorted(range(len(intervals)),
                   key=lambda i: (intervals[i][1], -intervals[i][0], i))
    chosen = []
    last_end = None
    for i in order:
        s, e = intervals[i]
        if last_end is None or s > last_end:
            chosen.append([s, e])
            last_end = e
    chosen.sort(key=lambda iv: (iv[0], iv[1]))
    return chosen
```
