I'll back the cache with a regular `dict`, relying on Python 3.7+ insertion-order semantics: the first key in iteration order is the least recently used. Every use (successful `get`, or any `put`) re-inserts the key at the end so it becomes most recently used. Eviction pops `next(iter(...))`, the oldest key. Updating an existing key pops-then-reinserts so both value and recency are refreshed.

```python
class LRU:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = {}

    def get(self, key):
        if key not in self.cache:
            return -1
        value = self.cache.pop(key)
        self.cache[key] = value  # move to most-recently-used
        return value

    def put(self, key, value):
        if key in self.cache:
            # refresh recency: remove so re-insert lands at the end
            self.cache.pop(key)
        elif len(self.cache) >= self.capacity:
            # evict least recently used (oldest insertion)
            oldest = next(iter(self.cache))
            self.cache.pop(oldest)
        self.cache[key] = value
```

This gives O(1) expected `get`/`put`, respects capacity exactly, and refreshes both value and recency on updates.
