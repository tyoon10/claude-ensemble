I build the encoder by scanning left to right and extending the trailing run when the
current character matches, otherwise starting a fresh `(char, 1)` group. Because the loop
never touches an initial element, the empty string naturally yields `[]`. Decoding expands
each `char * count`, guarding the count so a run always emits at least one character.

```python
def rle_encode(s):
    result = []
    for ch in s:
        if result and result[-1][0] == ch:
            prev_char, prev_count = result[-1]
            result[-1] = (prev_char, prev_count + 1)
        else:
            result.append((ch, 1))
    return result


def rle_decode(pairs):
    return "".join(ch * max(n, 1) for ch, n in pairs)
```
