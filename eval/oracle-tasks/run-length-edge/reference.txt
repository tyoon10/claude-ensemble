I build the encoder by scanning left to right and extending the trailing run when the
current character matches, otherwise starting a fresh `(char, 1)` group. Because the loop
never touches an initial element, the empty string naturally yields `[]`. Decoding just
expands each `char * count`, which makes zero counts contribute nothing and keeps the
round trip exact.

```python
def rle_encode(s):
    result = []
    for ch in s:
        if result and result[-1][0] == ch:
            prev_char, prev_count = result[-1]
            result[-1] = (prev_char, prev_count + 1)
        else:
            result.append((ch, 1))
    return result


def rle_decode(pairs):
    return "".join(ch * n for ch, n in pairs)
```
